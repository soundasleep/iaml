When the results of a {@model SelectWire} are evaluated, the resulting result set is not stored in memory. That is, if the underlying view changes (objects are inserted or removed), the users' view will be updated accordingly. For example, {@model DomainIterator#next} will skip over deleted results, possibly undesirably.

You are therefore advised to use the "hasNext" and "hasPrevious" {@model PrimitiveCondition}s that are provided in the {@model DomainIterator}, and connecting a {@model NavigateAction} as a fail wire if necessary.

A variety of database-independent query functions are provided:
Function Returns
matches(a,b) Performs a case-insensitive text search of a against the query b. Every word in b is matched, usually using a SQL keyword or function LIKE, against a.
now() Returns the current date and time.
