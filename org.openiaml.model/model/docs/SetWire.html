A {@model SetWire} will synchronise the {@model CanBeSynced} elements contained within each of the synchronised elements in a manner similar to a {@model SyncWire}, except that this {@model Wire} is unidirectional; if the target value changes, the source value will not change. For a {@model SetWire}, the default element created depends instead on the following semantics:
  • If both the source container and the target container are {@model InputForm}s, then the type of the default element will be a {@model Label}.
  • If the source container is a {@model DomainIterator} and the target container is a {@model InputForm}, {@model Email} or {@model IteratorList}, then the type of the default element will be a {@model Label}.
  • If the source container is a {@model VisibleThing} and the target container is a {@model Map}, then the type of the default element will be a {@model MapPoint}. \end{itemize} If a {@model DomainIterator} iterating over multiple results is connected to a {@model InputForm} with a {@model SyncWire} or {@model SetWire}, then the form will be populated in such a way to provide navigation over the data set. That is, {@model Button}s will be created within the form to control the cursor over the iterator.
