An IAML model instance may contain any number of {@model Frame}s. The {@event onInit} {@model Event} for an {@model InternetApplication} is triggered every time the {@model Frame} is rendered to a client.

The storage semantics for a {@model Frame} follow the storage semantics for its containing scope. For example, if the {@model Frame} is directly contained by a {@model Session}, then the storage semantics of the parent {@model Session} will be used; likewise, if the {@model Frame} is directly contained by an {@model InternetApplication}, then the storage semantics of the parent {@model InternetApplication} will be used.

A {@model Frame} may contain any number of {@model VisibleThing}s, and this set of elements defines the user interface for that {@model Frame}. All {@model VisibleThing}s may be rendered to a user. When a {@model VisibleThing} is rendered to a user, all children {@model VisibleThing}s must also be rendered in order according to their {@model VisibleThing#renderOrder}. A {@model VisibleThing} that is currently hidden must not be rendered until that {@model VisibleThing} is subsequently shown through the {@model VisibleThing#show} {@model Operation}. The {@model VisibleThing#name} propery of the {@model VisibleThing} may be used as a label for the rendered element.
