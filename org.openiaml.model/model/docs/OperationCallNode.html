An {@model ActivityOperation} may contain any number of {@model OperationCallNode}s. Each {@model OperationCallNode} must have at least one outgoing {@model ECARule}. When the execution flow arrives at an {@model OperationCallNode}, each outgoing {@model ECARule} will be executed sequentially, according to the {@model ECARule#priority} of each edge.

If the last {@model ECARule} executed from the {@model OperationCallNode} fails, the execution flow will follow all outgoing failure edges sequentially. Otherwise, the execution flow will follow all outgoing success edges sequentially.
