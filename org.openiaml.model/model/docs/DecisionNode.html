A {@model DecisionNode} can be used to control the flow of execution within a particular operation, and accepts any number of incoming {@model DataFlowEdge}s and {@model Condition}s, depending on the type of {@model DecisionNode}. A {@model DecisionNode} must have at least one failure edge and one success edge.

An {@model ActivityOperation} may contain any number of {@model DecisionNode}s. When the execution flow arrives at a {@model DecisionNode}, all incoming edges are passed to the particular node, and the decision is evaluated as either true or false. If the {@model DecisionNode} is considered to be false, the execution flow will follow all outgoing failure edges sequentially. If the {@model DecisionNode} is considered to be true, the execution flow will follow all outgoing success edges sequentially.

The {@model DecisionNode} element is based on the UML {@uml DecisionNode} model element.

true?

A {@model DecisionNode} {@model DecisionNode#name}d "true?" accepts any number of incoming {@model DataFlowEdge}s. This node is true if and only if all incoming values can be cast to the {@type boolean} type, and are all considered true through logical conjunction.

equal?

A {@model DecisionNode} {@model DecisionNode#name}d "equal?" accepts any number of incoming {@model DataFlowEdge}s. This node is true if and only if all incoming values can be cast to the {@type string} type, and all {@type string}-cast representations can be considered equal in terms of the fn:codepoint-equal {@model XQueryFunction}.

can cast?

A {@model DecisionNode} {@model DecisionNode#name}d "can cast?" accepts one incoming {@model DataFlowEdge} (source), and has one outgoing {@model DataFlowEdge} (target). This node is true if and only if the current instance from the source edge can be successfully casted to the data type specified by the target edge, as discussed in {@model CastNode}.
